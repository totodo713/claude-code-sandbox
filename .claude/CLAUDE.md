# 共通ルール

このファイルはプロジェクト横断で適用したい運用ルールを置く場所。
プロジェクト固有の情報 (このリポが何か、どうビルドするか等) はルートの
`CLAUDE.md` 側に書く。

## コミュニケーション言語

- **ユーザー向けの出力は日本語**で書く (応答本文、要約、コミットメッセージの
  日本語化が必要な場面など、人間が読む箇所すべて)。
- **思考とサブエージェントへのプロンプト・サブエージェントとのやり取りは
  言語自由**。英語の方が短く・正確に書ける場合は英語で構わない。
- コード内の識別子・コメント・ドキュメント・ログメッセージは既存ファイルの
  慣習に合わせる (英語ベースのプロジェクトに突然日本語コメントを混ぜない)。

## プランの出力先

設計プラン (plan mode の成果物を含む) はリポジトリ管理下の `.claude/plan/` に
Markdown で出力する。ファイル名は対象が分かる kebab-case とする
(例: `issue-1-corepack-dist-tag.md`)。harness の一時パス (`~/.claude/plans` 等) に
作った場合も、確定後にここへ複製する。
